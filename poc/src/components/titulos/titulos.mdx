# Títulos — hierarquia, uso e acessibilidade

Baseado em [designsystem.ms.gov.br/titulos](https://www.designsystem.ms.gov.br/titulos/). Entregue como
**classe CSS + tag HTML nativa** (`h1`–`h6` + `.ds-titulo.{nivel}`), sem componente JS — é tipografia, não
comportamento.

## Hierarquia (H1–H6)

| Nível | Uso | Peso | Tokens |
|---|---|---|---|
| H1 | Título principal da página (único) | 700 (Bold) | `--titulos-h1-font-size` (32px) |
| H2 | Seção principal | 700 | `--titulos-h2-font-size` (24px) |
| H3 | Sub-seção | 600 | `--titulos-h3-font-size` (20px) |
| H4 | Agrupamento secundário | 600 | `--titulos-h4-font-size` (18px) |
| H5 | Título contextual | 600 | `--titulos-h5-font-size` (16px) |
| H6 | Rótulo/microtítulo | 400 | `--titulos-h6-font-size` (14px) |

> A escala typography.json só tem 3 pesos (400/600/700) — por isso H2–H5 compartilham 600 em vez da
> distinção Medium/SemiBold do guia oficial.

## Regra de hierarquia

- **Um único H1 por página.**
- Não pular níveis (H1 → H3 sem H2) — a hierarquia visual deve refletir a estrutura do documento, não o
  tamanho desejado.

## Acessibilidade (eMAG + WCAG 2.1 AA)

| Categoria | O que verificar | Como |
|---|---|---|
| **Leitor de tela** | Navegação por níveis de título (`H` no NVDA, rotor no VoiceOver) depende de ordem correta, sem saltos | NVDA / VoiceOver |
| **Contraste** | Texto ≥ 4.5:1 | `--titulos-color` = `--color-neutral-900` sobre fundo claro passa |
| **Zoom** | Legível a 200% | Usa `rem`/`px` da escala, sem `white-space: nowrap` |

Sem token de cor/peso isolado substitui a marcação semântica — o nível continua sendo a tag HTML
(`<h1>`...`<h6>`), nunca apenas uma classe CSS num `<div>` ou `<span>`.
