# Listas — uso e acessibilidade

Baseado em [designsystem.ms.gov.br/listas](https://www.designsystem.ms.gov.br/listas/).
Classe CSS + marcação semântica (`<ul>`/`<ol>`/`<li>`) — nunca `<div>` simulando item de lista.

## Variantes

| Variante | Classe | Uso |
|---|---|---|
| Não ordenada | `.ds-lista` em `<ul>` | Itens sem relação de ordem/prioridade |
| Ordenada | `.ds-lista` em `<ol>` | Passos sequenciais, ranking |
| Com ícone | `.ds-lista.com-icone` | Substitui marcador padrão por indicador de status |
| Compacta | `.ds-lista.compacta` | Dentro de cards, accordions, modais |

## Espaçamento

- `--listas-item-gap` entre itens (8px), `--listas-compact-item-gap` (4px) na variante compacta.
- `--listas-indent` (24px) de recuo na variante padrão.
- `--listas-marker-color` colore o marcador (`::marker` ou `::before`) sem depender só de cor pra
  diferenciar — o ícone `✓` reforça o significado, não é decoração isolada.

## Acessibilidade (eMAG + WCAG 2.1 AA)

| Categoria | O que verificar | Como |
|---|---|---|
| **Leitor de tela** | Anuncia "lista com N itens" | Exige `<ul>`/`<ol>`/`<li>` reais, nunca `<div>` |
| **Contraste** | Marcador e texto ≥ 4.5:1 | `--listas-marker-color` = `primary-600` sobre branco passa |
| **Estrutura** | Itens em ordem paralela, sem item único | Revisão de conteúdo, não só CSS |
