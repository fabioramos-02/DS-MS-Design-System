# Formulários — uso e acessibilidade

Baseado em [designsystem.ms.gov.br/formularios](https://www.designsystem.ms.gov.br/formularios/).
**Não é um componente de input novo** — Input, Label, Selection e Button já existem
(`Componentes/Input`, `Componentes/Label`, `Componentes/Selection`, `Componentes/Button`). Este padrão
documenta só **composição**: grid, seções e bloco de ações.

## Classes de composição

| Classe | Função |
|---|---|
| `.ds-form-section` | Container de seção (fieldset), fundo sutil + espaçamento interno |
| `.ds-form-grid` | Grid responsivo (2–3 colunas, `auto-fit minmax(220px,1fr)`) |
| `.ds-form-help` / `.is-error` / `.is-success` | Texto de ajuda/erro/sucesso abaixo do campo |
| `.ds-form-required` | Asterisco de campo obrigatório |
| `.ds-form-actions` | Bloco de ações: primária à direita, cancelar/secundárias à esquerda |

## Regras de layout

- Máximo 2–3 colunas por seção — prioriza leitura vertical.
- Uma única ação primária por formulário.
- Estado de erro: borda de erro (já em `input.css` via `.is-error`) + texto explicativo —
  **nunca só a cor** indica o problema.
- Estado de sucesso: mesma lógica, com `.ds-form-help.is-success`.

## Acessibilidade (eMAG + WCAG 2.1 AA)

| Categoria | O que verificar | Como |
|---|---|---|
| **Rótulos** | `label[for]` associado ao `id` do campo | Inspeção de markup |
| **Ordem de tabulação** | Sequência lógica, sem `tabindex` positivo | Navegação só com `Tab` |
| **Erro anunciado** | Leitor de tela lê a mensagem de erro ao focar o campo | `aria-describedby` apontando pro `.ds-form-help` |
| **Contraste** | Texto de ajuda/erro ≥ 4.5:1 | `--formularios-error-color` (`red-700`) sobre branco passa |
| **Obrigatoriedade** | Asterisco não é o único sinal — usar também `aria-required="true"` | Inspeção de markup |
