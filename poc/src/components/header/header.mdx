# Header — Web Component, uso e acessibilidade

`<ms-header>` é o **primeiro Web Component** do DS-MS (decisão de arquitetura em [`docs/01-arquitetura.md`](../../../../docs/01-arquitetura.md)): componentes interativos (menu mobile, foco gerenciado) ganham comportamento centralizado num custom element, em vez de cada time reimplementar o JS do menu hambúrguer.

## Por que Web Component, e não só CSS+HTML
O header tem **comportamento** (abrir/fechar menu em mobile, gerenciar `aria-expanded`). Se isso fosse só CSS+HTML, cada time em PHP/Python/JS teria que escrever esse JavaScript de novo — exatamente o problema que motivou a POC. Centralizando em `<ms-header>`, qualquer stack só cola a tag:

```html
<script type="module" src=".../ds-sis.js"></script>
<ms-header secretaria="SETDIG" version="v2.0.0"></ms-header>
```

## Decisão técnica: Light DOM, não Shadow DOM
`ms-header.js` sobrescreve `createRenderRoot()` para retornar `this` — o componente renderiza em **light DOM**. Isso permite que o CSS global do design system (`header.css`, equivalente ao `components.css` da raiz) estilize o componente normalmente, sem duplicar tokens dentro de um Shadow DOM isolado. Trade-off consciente: perde-se o encapsulamento de estilos do Shadow DOM, mas mantém-se **uma única fonte de CSS** para todo o DS — consistente com a arquitetura de tokens.

## Acessibilidade (eMAG + WCAG 2.1 AA)
- `role="banner"` no `<header>` — marca a região como cabeçalho da página para leitores de tela.
- Botão de menu com `aria-expanded` (true/false) e `aria-controls="ms-header-nav"` — leitor de tela sabe o que o botão controla e seu estado atual.
- `aria-label` do botão muda entre "Abrir menu" / "Fechar menu" — nunca um ícone sozinho sem texto acessível.
- Links de navegação com `:focus-visible` — outline branco visível sobre o fundo azul (contraste).
- Versões **Mobile fechado** e **Mobile aberto** abaixo demonstram os dois estados do menu sem depender da largura real do iframe do Storybook (prop `force-mobile`); em produção, o mesmo comportamento dispara via `@media (max-width: 768px)`.
