# Search — uso e acessibilidade

Campo de busca compacto (`.search`), usado em headers e listagens. Atualmente é **CSS+HTML puro** — sem comportamento JS (sem autocomplete/sugestões). Se ganhar essas funções, vira candidato a Web Component (`<ms-search>`), igual ao [Header](?path=/docs/componentes-header--docs).

## Acessibilidade
- `role="search"` no `<form>` — leitores de tela anunciam a região como busca.
- `aria-label="Buscar"` no `<input>` — supre a falta de `<label>` visível.
- Ícone é `aria-hidden="true"` — decorativo, não duplica informação para leitor de tela.
- Foco visível via `:focus-within` no container (borda + halo), não só no input.

## Código

```html
<form class="search" role="search">
  <svg aria-hidden="true">...</svg>
  <input type="search" placeholder="Buscar" aria-label="Buscar" />
</form>
```
